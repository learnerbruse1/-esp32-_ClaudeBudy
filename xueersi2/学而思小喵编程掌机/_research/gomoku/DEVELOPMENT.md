## 一、问题分类汇总

### 🔴 驱动层问题

| # | 问题现象 | 根本原因 | 最终方案 | 避坑提示 |
|---|---------|---------|---------|---------|
| 1 | 画面整体斜纹撕裂，完全不可读 | LVGL 软件旋转 + ST7735 硬件旋转**双重生效**，方向叠加导致错位 | **仅用 LVGL 软件旋转**（`swap_xy=true, mirror_x=true`），ST7735 端不做任何 `swap_xy`/`mirror` 配置 | 嵌入式 LCD 旋转**只在一层做**，推荐 LVGL 层（灵活性高） |
| 2 | 设置 `gap` 非零值后画面斜纹 | ST7735 驱动与 LVGL 的 `set_gap()` 不兼容，修改显存地址偏移导致数据线错位 | **永久 `gap=0`**，放弃硬件级帧同步 | LVGL 端口层 `gap` 参数不是所有屏都支持，默认 0 最安全 |
| 3 | LEDC 结构体编译警告 `-Wmissing-field-initializers` | ESP-IDF v5.4 新增 `deconfigure`/`sleep_mode`/`flags` 字段 | 显式初始化所有字段为 0/false | IDF 版本升级后检查新增的结构体成员 |
| 4 | 蜂鸣器播放后持续微响 | PWM 播放结束后未关断占空比 | 每次 `tone()` 结束调用 `ledc_set_duty(0)` | 无源蜂鸣器必须占空比归零才能完全静音 |

### 🟡 LVGL UI 适配问题

| # | 问题现象 | 根本原因 | 最终方案 | 避坑提示 |
|---|---------|---------|---------|---------|
| 5 | 喇叭图标只显示约 1/4 | `lv_layer_top()` 上状态栏先创建、全宽覆盖，同层渲染顺序导致图标被遮挡 | 将图标改为**状态栏的子对象**（`lv_obj_create(statusbar_obj)`），保证 Z 序正确 | 同一 Layer 上多个控件时注意创建顺序 ≠ 渲染顺序，父子关系更可靠 |

| 9 | 坐标标签（行列号）无法容纳 | 14px 字体在 7px 格距下无法放置单行/列标签 | **取消坐标标签**，仅靠棋盘星位点定位 | 物理尺寸冲突无解时果断砍功能 |
| 10 | 标题/设置等页面重复创建未释放 | `create_xxx_screen()` 每次新建 `lv_obj_create(NULL)`，旧屏未显式删除 | 当前不影响使用（内存充足），Phase 12.5 待修 | 多屏架构需管理生命周期，LVGL 的 `lv_screen_load` 不会自动删除旧屏 |

### 🟠 交互逻辑问题

| # | 问题现象 | 根本原因 | 最终方案 | 避坑提示 |
|---|---------|---------|---------|---------|
| 11 | 胜利判定错误（冲四即判胜） | 原 Pela 逻辑 `h[player].pv >= H4` 将冲四（2047）当作五连 | 重写为**方向连续计数法**，从落子点向 4 方向统计连续同色 ≥ 5 | 移植 AI 引擎时必须验证胜负判定逻辑与目标规则一致 |

| 13 | B 短按音效切换无效 | `g_sound_on`（UI 状态）和 `g_audio_enabled`（硬件状态）分离，设置变更时未同步 | 在 `ui_settings_change_val()` 中增加 **`audio_set_enabled()` 调用** | 全局状态必须保持单一真值源（Single Source of Truth） |

| 15 | `sound_icon` 编译重定义 | 模块级 `static` 变量在两处函数外重复声明 | 删除重复声明，保留一处 | 使用 `grep` 检查所有 `static` 变量是否唯一 |

## 二、核心开发注意事项

### 1. ST7735 小屏横屏适配核心要点

```
问题本质：物理 128×160 竖屏 → 逻辑 160×128 横屏需要旋转 90°

方案对比：
  ❌ 硬件旋转    — 修改 ST7735 寄存器（MADCTL），简单但之后所有坐标需换算，LVGL 端口冲突
  ❌ 双重旋转    — 硬件+软件同时做，方向叠加导致无法调试的撕裂
  ✅ 仅 LVGL 旋转 — swap_xy=true + mirror_x=true，硬件端完全不做旋转

关键配置（lvgl_port_display_cfg_t）：
  .rotation = { .swap_xy = true, .mirror_x = true, .mirror_y = false }
  .hres = 160, .vres = 128  （注意：横屏后宽160高128）

显存注意事项：
  - buffer_size = 160 × 128 = 20480 字节（双缓冲 ×2 = 40KB）
  - 物理分辨率 128×160，逻辑分辨率 160×128，两者完全对调
  - 硬件不改变扫描方向，LVGL 通过软件算法重映射每个像素
```

### 2. LVGL 9.2 在 160×128 极低分辨率下的布局技巧

```
字体选择：
  - 仅启用 lv_font_montserrat_14（14px），禁用其他字体节省 Flash
  - 14px 是可读性与空间的最佳平衡点：中文约 7 字/行，英文约 14 字符/行

布局策略：
  - 优先使用绝对定位（lv_obj_set_pos），Flex/Grid 布局在小屏上留白过大
  - 控件间距以 1px 为单位微调，不能依赖默认 padding（通常 4-8px）
  - 所有控件显式设置 pad_all=0, border_width=0, radius=0 去冗余

色彩策略：
  - 深海军蓝主题（#002A58），在 ST7735 上对比度优于纯黑
  - 选中态用反色（白底蓝字），比单纯改色更醒目
  - 木色棋盘（#B8A080）配深棕网格线（#6B5335），模拟真实棋盘质感

常见陷阱：
  - lv_obj 默认有 2-4px 的 padding，不显式清零会导致布局溢出
  - lv_label 中文需要字体包含对应 glyph，未包含的字显示为方框
  - 双缓冲模式下 buffer_size 必须是 hres×vres 的整数倍
```

### 3. 实体按键 + 焦点导航的交互设计

```
设计原则：
  - 单选即时生效：方向键切换选项后立即更新 UI 状态，无需按确认
  - 焦点与视觉绑定：焦点行=反色高亮，非焦点行=下划线标记当前值（双视觉态）
  - 全局键位一致：所有页面 A=确认/前进，B=返回/取消（肌肉记忆）

B 键双功能（短按/长按）实现：
  1. ISR 记录下降沿时刻
  2. input_poll() 检测到 B 键后延时 510ms 再采样
  3. 仍为低电平 → 长按（BTN_B_LONG），已恢复高电平 → 短按（BTN_B）
  4. 长按后等待释放再返回，避免长按被误识别为短按

焦点导航实现：
  - 设置页 4 行（3 选项 + START）：↑↓切换行，←→切换值
  - 游戏页 15×15 网格：↑↓←→移动光标，边界限制在 0-14
  - 光标坐标持续跟踪，不依赖 UI 层反向查询
```

### 4. 全局状态跨页面同步原则

```
核心原则：Single Source of Truth（单一真值源）

音效状态同步链：
  g_sound_on (ui.cpp) ──→ g_items[2].value (UI 显示)
                       ──→ audio_set_enabled() (硬件 PWM)
                       ──→ ui_update_sound_icon() (图标更新)

状态变更点：
  - 设置页 ←→键：ui_settings_change_val() → 同步更新上述三者
  - 游戏内 B 短按：ui_sound_toggle() → 原子切换上述三者
  - 回到标题：ui_show_title() → 强制复位 g_sound_on=true

❌ 错误做法：在多个调用点分别设置 g_sound_on 和 g_audio_enabled
✅ 正确做法：所有变更集中到 ui_sound_toggle()，所有复位集中到 ui_show_title()
```

## 三、调试方法论

### 嵌入式 UI 项目标准排查顺序

```
优先级从高到低：

1. 先硬件、后软件
   - 用示波器/逻辑分析仪确认 SPI 波形正常
   - 用万用表确认 GPIO 电平正确
   - 确认供电电压（3.3V）稳定

2. 先驱动、后 UI
   - 确认 ST7735 初始化序列完整（Gamma/电源/帧率寄存器）
   - 先用纯色填充测试（红/绿/蓝/白），验证显存→屏幕映射正确
   - 确认 LVGL flush_cb 被正常调用，帧率 ≥ 30fps

3. 单变量迭代调试
   - 每次仅改一个参数（如 swap_xy、mirror_x、gap），烧录验证
   - 记录每次改动和对应的显示效果（拍照或文字描述）
   - 定位到最小可复现配置

4. 分层隔离
   - 驱动问题：用 esp_lcd 原语直接写像素，绕过 LVGL
   - LVGL 问题：用 lv_demo_widgets 验证框架功能
   - UI 问题：用串口日志替代 UI 元素验证逻辑
```

### 常见故障快速定位

| 故障 | 可能原因 | 排查步骤 |
|------|---------|---------|
| **花屏**（随机色块） | 显存地址错位、SPI 时序错误 | 1. 检查 buffer_size 匹配 2. 检查 SPI 时钟 ≤ 40MHz 3. 单色填充测试 |
| **斜纹撕裂** | 双旋转、gap 非零、buffer 未对齐 | 1. 确认仅一层旋转 2. 设 gap=0 3. 检查 hres/vres |
| **控件重叠/错位** | LVGL 布局计算错误、padding 未清零 | 1. 逐个控件设背景色定位 2. 检查 pad_all=0 3. 用绝对坐标替代自动布局 |
| **按键无响应** | ISR 未安装、GPIO 冲突、防抖过严 | 1. 用 `gpio_get_level()` 读原始电平 2. 示波器确认波形 3. 降低防抖阈值 |
| **音效无声** | LEDC 未初始化、占空比为零、引脚冲突 | 1. 用 `ledc_get_duty()` 确认占空比 2. 万用表测 GPIO14 电压 3. 检查 `g_audio_enabled` 状态 |
| **AI 不落子** | K 表未生成、PSRAM 分配失败、搜索超时 | 1. 串口打印 `evaluate_init()` 完成日志 2. 检查 PSRAM 剩余量 3. 增加超时时间 |
